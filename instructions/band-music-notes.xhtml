Band Music Notes
  1. Front matter includes a title page; copyright page; Foreword; and lists of Abbreviations.
    • The "Sample Abbreviations" explains the layout of each item in the works list of each entry.
  2. Back matter includes five Appendices:
    1. Lists of Contributors.
      • Set the individuals' list as a two-column <table>; Tag the names in the left column as <rilmPerson>; tag the content of the right column as <placeName> if there is a comma and no parentheses, and as <orgName> if there is no comma OR there are parentheses.
      • Tag the Organizations list as a <list>.
    2. Key to Publishers. Set as a <list> and mark the parts as follows:
      [ACA American Composers Alliance, 170 W. 74th St., New York, NY]
      <item><abbr>ACA</abbr> <orgName>American Composers Alliance</orgName>, <address><street>170 W. 74th St.,</street> <placeName><settlement>New York</settlement>, <region>NY</region></placeName></address></item>
    3. Key to Record Companies. Tag the same as Key to Publishers.
    4. List of Titles. Do not create a TEI file. The content will be found in the individual entries.
    5. Vita. Tag as a series of <p> elements.
  3. Entries include a heading, subheading, body and works list.
    • Use @rend="noMarker" for the <birth> and <death> elements and do not omit any text.
    • The work lists are the main part of each entry. Mark up as a <div type="works"> and use <list> and <item> as usual.
      • Include only the italic title and what is in parentheses in the <bibl>.
      • Tag what follows "Pub=" as <publisher><abbr>; tag the year or years as <date>.
      • Tag what follows "Rec=" as <idno type="label">.
      • Tag what follows the recording number as <orgName type="performing">
      • Enclose the following paragraph in a <note> element. If there is an author name in parentheses at the end of the note, tag as <persName>, and <affiliation>.

Please refer to the sample files.
