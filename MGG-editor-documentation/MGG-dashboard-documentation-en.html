Documentation for the MGG User Dashboard


Note: The editorial system is Web-based, and requires only a Web browser and an internet connection. The system supports recent versions of Google Chrome and FireFox. We strongly recommend using Chrome for all work in the system.

General instructions

Our User Dashboard allows authors, translators, and other contributors to MGG access to the articles to which they have been assigned.

“Articles” on the Dashboard are sometimes whole MGG-Online articles; sometimes they are only parts of articles, which have been split up for easier editing. When they are published, split articles will appear together as one unbroken article.

There are six lists of articles accessible from the Dashboard: View all articles, Articles with recent messages, Open articles, Assigned articles, Collaboration, and Past assignments. Clicking on the buttons at the top left of the screen will display the appropriate list. When you open the Dashboard up, it will default to Open articles.

List of Articles

For articles that have been split, the Section column shows which part you have been assigned to. For articles that have not been split, it will always show “Part 0”.

There are two kinds of message that the system holds pertaining to an article. Public messages (and notes) are viewable to all who have been assigned to the article, as well as all MGG-Online editors. A count of those messages, since the article was last published or translated, appears on the Dashboard in the General column. Private messages can only be seen by the sender and the receiver. The Private column shows a count of private messages to which you have sent or received.

Click on a row to see more information about an article, or (if in you have access) to edit it. On the right, you see buttons to complete various actions, and two boxes (Details and Document History) with further information about the article.

Buttons

When you select an article in the dashboard, buttons will appear under the title of the article on the right side of the screen. The buttons vary depending on whether you have access to that particular article:

  • Submit will give access to the article back to the editor.
  • In the case that there are more than one user (author or translator) working on one part of an article together, Collaborate allows one user to give access to another user with the same role. Access can then be passed back and forth indefinitely.
  • Edit opens the article in the Editor.
  • View opens the article for viewing or printing only.
  • Add Note allows you to write a note or message relating to the selected article. A message is addressed to a particular user or editor in the system. A note is not addressed to anyone in particular. If a message is marked Private, only the writer and the recipient of that note will be able to see it.
  • Approve allows a user (author or editor) to approve an article for publication.

You can only edit articles you have been given permission to access. Only one user can have access to an article at a time. You can view and add notes to any article to which you have been assigned.


Document History

To read notes and messages relating to an article, click See more at the bottom of the Document History section.
