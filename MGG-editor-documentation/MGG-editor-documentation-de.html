Dokumentation für FontoXML/MGG


Das Editorial System ist web-basiert und erfordert lediglich einen Web Browser und Internet-Zugang. Das System unterstützt aktuelle Versionen von Firefox und Google Chrome. Wir empfehlen jedoch dringend, das Tool via Chrome zu öffnen und zu nutzen. Dazu müssen Sie sich eine aktuelle (kostenfreie) Version von Google Chrome auf den Rechner laden.

Allgemeine Hinweise

Formale Auszeichnung der Texte (Tagging)

Wenn Sie mit Textverarbeitungsprogrammen wie Microsoft Word oder Google Docs vertraut sind, werden Sie mit einigen Funktionen von FontoXML intuitiv umgehen können. Andere wurden speziell für komplexe Dokumente wie jene der MGG Online entwickelt.

Wenn Sie einen bestimmten Absatz oder Artikel anklicken, erscheint eine Markierung, die zeigt, welcher Text wie ausgezeichnet (getaggt) wurde. Diese Auszeichnungen (tags) erscheinen in den Farben blau, grau, rot und gelb, je nachdem welche tags verwendet wurden.

Gelbe Auszeichnungen ohne Einrahmung sind Formatauszeichnungen (wie z.B. »Text in Anführungszeichen«), ohne dass inhaltlich bestimmt ist, um was es sich bei diesem ausgezeichneten Text handelt (»Text in Anführungszeichen« kann sein: Incipit, Zitat o.a.). Bitte korrigieren Sie diese Tags wenn möglich.

Rot unterstrichene Textstellen zeigen einen Kommentar der Redaktion an; sie werden nicht veröffentlicht. Der Kommentar wird sichtbar, wenn man auf den rot unterstrichenen Text klickt.

Standort/Position im Text

Am unteren Ende des Bildschirms befindet sich ein Display, welches die Position im Dokument anzeigt. Wenn der Cursor in den hervorgehobenen Text platziert wird, zeigt das display an, mit welchem Tag die hervorgehobene Stelle ausgezeichnet wurde. Im Beispiel unten befindet sich der Cursor im Birth-Tag.

Menüs der rechten Bildschirmseite

Auf der rechten Bildschirmseite befinden sich drei Tabs: Properties, Structure und Source:

  • Structure: Dieser Tab zeigt die Hauptabschnitte des Artikels an. Der Name des Abschnitts, in welchem sich der Cursor befindet, erscheint in Fettschrift. Dieser Tab ist hilfreich, wenn man schnell von einem Abschnitt des Textes zum nächsten springen will
  • Properties: Die Merkmale des verwendeten Tags. Einige Tags haben unterschiedliche Eigenschaften. So kann man z.B. die Sprache der Tags genau bestimmen: für einen Satz, der mit dem Tag foreign ausgezeichnet wurde, kann die genaue Sprache in dem language-property gewählt werden. Man muss auf das Pluszeichen ()klicken, dann die Sprache auswählen und schließlich save properties klicken. Um die Sprache eines ganzen Artikels zu bestimmen, setzt man den Cursor an den Anfang des Textes (er erscheint horizontal), dann klickt man auf Text im Property-Tab. Sodann ändert man die Sprache und sichert mit Save properties.

Abschnitte und Kapitel

Einen neuen Abschnitt erzeugen

Um einen neuen Abschnitt zu erzeugen, klicken Sie am Ende eines bestehenden Abschnitts oder einer Kapitelüberschrift auf Enter oder Return. Wenn Sie einen neuen Absatz in einer Liste (Werklist, Literaturliste) erzeugen wollen, klicken Sie ebenfalls einmal auf Enter, zweimal auf enter, wenn Sie einen neuen Textabschnitt nach dieser Liste erzeugen wollen.

Ein Kapitel erzeugen

Um ein Kapitel zu erzeugen, müssen Sie zunächst den Cursor irgendwo in einem bereits bestehenden Kapitel platzieren und den entsprechenden Typ von Kapitel im Structure-Menü des Insert-Menüs auswählen. Das neue Kapitel wird am Ende des Kapitels, in welchem sich der Cursor befindet, erscheinen.

Ein Unterkapitel erzeugen

Um ein Unterkapitel zu erzeugen, müssen Sie zunächst ein neues Kapitel unterhalb des Kapitels (also nach dem Kapitel), in welchem das Unterkapitel erscheinen soll, erzeugen. Dann öffnen Sie das Structure-Menü, wo Sie ein Auslassungs-Icon sehen (). Sie klicken auf das Icon. Um das neue Kapitel zu einem Unterkapitel (des obenstehenden Kapitels) zu machen, klicken Sie auf Move right.

[[Wenn in dem Kapitel, welches Sie zu einem neuen Unterkapitel machen wollen (ist das richtig übersetzt?)]], Text vorhanden ist, müssen Sie das neue Kapitel oberhalb des existierenden Kapitels erzeugen und das bestehende Kapitel in ein Unterkapitel des neuen Kapitels mit Move right umwandeln.

Sie können den Typ jedes Kapitels in dem properties-Menü ändern.

Zählung der Kapitel ändern

Um die Zählung eines Kapitels zu ändern oder hinzuzufügen, muss man das number-Feld im properties-Menü benutzen. Man klickt die Überschrift an, geht dann in die Fußleiste auf den ( letzten) Button Section. Dann muss im number-Feld des properties-Menüs die Ziffer entsprechend geändert werden.

Inhaltsverzeichnisse updaten

Wenn Änderungen in der Kapitel-Organisation der Sachteil-Artikel erfolgen, also z.B. neue Kapitel eingefügt und bereits vorhandene gelöscht oder umnummeriert werden, müssen auch die Inhaltsverzeichnisse dementsprechend geändert werden. Diese Inhaltsverzeichnisse sind als Listen angelegt (vgl. B.2.2; dort auch Näheres zum Update).

Um einen Link zwischen einer Kapitelüberschrift im Inhaltsverzeichnis und der entsprechenden Überschrift im Artikel selbst zu setzen, öffnen Sie das Insert-Menü und wählen den tag link, {Local Link. Wenn das Dialog-Feld erscheint, schreiben Sie die Überschrift in das Feld Filter by Text. Sobald die entsprechende Kapitelüberschrift unter Items in this document erscheint, wählen Sie es aus und klicken Insert.

Das Lösch-Icon

Mit dem Lösch-Icon (), das sich in FontoXML oben links befindet, können Sie Tags löschen. Setzen Sie den Cursor auf eine beliebige Stelle im Dokument, und der Tag, auf dem Sie sich befinden (wie im Properties-Tab angezeigt) wird gelöscht. Es wird kein Text gelöscht - nur die Auszeichnung. Wenn mehrere Tags übereinander erscheinen, müssen Sie zuerst alle Tags auf der untersten Ebene löschen, bevor Sie die Tags, die sie enthalten, löschen.

Masken (Templates)

Für die Aufnahme von Literaturtiteln in den Literaturverzeichnissen gibt es Masken (Templates), in die Sie direkt hineinschreiben können. Die Tags sind dort bereits vorhanden:

Sie finden diese Masken im Menü MGG/Bibliogaphy/Bibliographic Citation. Dort sind drei Grundmasken für die Aufnahme von Literaturtiteln aufgeführt: Monograph title template, Periodical article Template, Collection article template.

Schreiben Sie direkt in die grauen Felder um sie zu vervollständigen bzw. zu füllen. Jedes Feld kann auch leer gelassen werden: so wird nichts im Text erscheinen. Jeder Text oder jede Interpunktion außerhalb der Felder muss gelöscht werden, wenn Sie nicht wollen, dass etwas davon im Text erscheint. Wenn Sie zusätzliche Angaben im Literaturtitel haben, die nicht durch die in den Masken vorhandenen Felder abgedeckt sind, können Sie diese einfach zwischen die grauen Felder schreiben (und dann taggen).

FontoXML - Menüs

Das MGG-Menü: Auszeichnung (Taggen) von Textelementen (Wörter, Sätze, Abschnitte)

Mit diesem Menü können die Texte bzw. die einzelnen Textbausteine augezeichnet werden (getaggt werden). Es enthält die am häufigsten benutzten Tags zur Auszeichnung von MGG-Texten in den sechs Menüs: Names, Quotation, Bibliography, Works, Titles und Formatting.

Namen (Names)

Person

Mit Person werden Namen von und Angaben über Personen ausgezeichnet und formatiert. Er wird verwendet

  1. um den Anfang von Personenteil-Artikeln über Personen, sowie Abschnitte innerhalb Personenteil-Artikel über Familien auszuzeichnen
  2. um einen Namen auszuzeichnen, der im Fließtext erscheint und mit biographischen Angaben wie z.B. Geburts- oder Sterbedatum versehen ist.

Der gesamte Eintrag (inkl. Lebensdaten, Beruf u.a.) wird zunächst mit Person ausgezeichnet.

Der Personenname wird dann mit Personal name ausgezeichnet. Weitere auszuzeichnende Differenzierungen innerhalb dieses Tags sind:

Vorname , auszuzeichnen mit Forename

Nachname , auszuzeichnen mit Surname,

Namenszusatz , auszuzeichnen mit Surname prefix

Generation , auszuzeichnen (wie »d.J.«, »ll.«, »Jr.«) mit Generational name component

Funktion , auszuzeichnen mit Role name

Zusätzliche Namen, auszuzeichnen mit Additional name.

Abweichende Schreibweisen, Romanisierungen oder andere Formen, auszuzeichnen mit Variant personal name

Spitznamen , auszuzeichnen mit Nick name

Name des Vaters , auszuzeichnen mit Patronym,

Beiname , auszuzeichnen mit Epithet.

Innerhalb des Tags Person werden dann Lebensdaten und andere Angaben ausgezeichnet:

Geburt mit Birth, nochmals differenzieren in Datum Date und Ort Place

Taufe mit Baptism

Floruit mit Floruit

Beruf mit Occupation

Tod mit Death, nochmals differenzieren in Datum Date und Ort Place

Begräbnis mit Burial.

Angaben zur Geburt und zum Tod

Wie oben bereits gezeigt, müssen allen Angaben zur Person innerhalb des Person-Tags weiter differenziert und dementsprechend ausgezeichnet werden. Zeichnen Sie alle Angaben zur Geburt einer Person mit Birth, zum Tod mit Death, alle Angaben zur Taufe mit Baptism und alle Angaben zum Begräbnis mit Burial aus. Differenzieren Sie innerhalb dieser Tags noch einmal nach Datum und Ort und zeichnen Sie entsprechend mit Date und Place name aus. Sie brauchen kein * oder † bei Birth oder Death einzufügen. Vorhandene Sternchen oder Sterbekreuze müssen entfernt werden.

Namen von Organisationen

Der Name einer Organisation - vor allem wenn sie Thema des Artikels ist - kann mit Organization name oder Variant organization name ausgezeichnet werden. Konzertierende Organisationen wie Orchester oder Rockgruppen können mit Performing group ausgezeichnet werden und werden kursiv wiedergegeben.

Ortsnamen

Ortsnamen, wenn sie in Person oder Performance (im Menü Works) erscheinen, können mit Place name ausgezeichnet werden.

Zitierung/Anführungszeichen (Quotation)

In diesem System verwenden Benutzer keine Anführungsstriche; stattdessen werden sie bei der Auszeichnung des Textes von einigen Tags erzeugt und erscheinen damit automatisch im veröffentlichten Text.

Zitate

aus (Sekundär-)Quellen werden mit dem Tag Quotation ausgezeichnet. Folgt auf das Zitat ein

Quellenhinweis

, zeichnen Sie zunächst Zitat und Quellenhinweis zusammen mit cited quotation aus. Dann wird das Zitat nochmals als citation und der Quellenhinweis mit Reference getaggt. Wenn der Quellenhinweis in einer Literaturliste im Artikel erwähnt wird, verwenden Sie (Insert)/Link/Local Link ansttel des Tags Reference.

Für sogenannt verwenden Sie den Tag So-called, zu finden im Menü Formatting.

Für Incipit-Titel verwenden Sie Incipit title aus dem Titel-Menü. Für Titel innerhalb anderer Titel verwenden Sie einfach ein title innerhalb des anderen; der "Kind"-Titel wird automatisch in Anführungsstrichen wiedergegeben.

Bibliographien und Werkverzeichnisse (Bibliography und Works)

Mit den Menüs Bibliography und Works werden Werkverzeichnisse, Bibliographien und ähnliche Verzeichnisse und Tabellen erstellt oder ausgezeichnet.

Um einen bibliographischen Titel oder einen Werktitel innerhalb dieser Verzeichnisse zu erstellen, fügen Sie den Tag Bibliographic citation oder Works citation hinzu und ergänzen Sie die Informationen innerhalb dieser Felder. Es gibt drei Masken (Templates) für bibliographische Titel, die in dem Menu MGG/Bibliography/Bibliographic citation verfügbar sind (s.o. unter Punkt A 5, Templates).

Autor Zeichnen Sie den Autornamen mit dem Tag Author aus, zusammen mit allen Namensteilen für Autoren (forename, surname etc.). Hat ein Autor eine besondere Funktion, wie etwa Librettist oder Lyriker, zeichnen Sie ihn ebenfalls einfach mit Author aus. Ist der Autorname abgekürzt, taggen Sie separat nochmals mit Abbreviation. Unterscheiden können Sie ebenfalls zwischen author-primary und Author-secondary.

Herausgeber Verwenden Sie Editor für Herausgeber. Verwenden Sie ggf. andere Tags, wie Translator, Compiler oder Illustrator; verwenden Sie Editor für jede andere Person, deren Beitrag zu dem Werk von untergeordneter Bedeutung ist. Unterscheiden Sie nochmasl zwischen Editor-primary and Editor-secondary.

Titel: Zeichnen Sie den Titel mit dem korrekten Titeltyp aus (im Menü Titel). Ist ein Titel sekundär (unselbstständig), d.h. handelt es sich um einen Titel, in welchem der primäre Titel erscheint (Anthologien, Kongressberichte, Festschriften, Periodika etc.), muss er mit »secondary« ausgezeichnet werden. Er erscheint dann nicht kursiv, sondern gerade.

Erscheinungsort Mit dem Tag Publication place wird der Erscheinungsort ausgezeichnet. Der Tag findet sich sowohl im Menü Works als auch im Menü Bibliography.

Datum : Mit dem Tag Date und seinen Varianten können ausgezeichnet werden:

- Erscheinungsdatum publishing date, zu finden im Menü Bibliography/Date und im Menu Works/Date. Hochgestellte Ziffern (Ausgabe) können erzeugt werden, indem man die Zahl mit dem Tag MGG/Bibliography/Edition (superscript) auszeichnet.

- Kompositionsdatum Composition date, zu finden im Menu Works/Date. Das Datum muss zuvor in Klammern gesetzt werden.

Aufführungsdaten : Eine Erstaufführung wird mit dem Tag First perfomance ausgezeichnet (zu finden im Menü Works/Performance), eine Uraufführung mit dem Tag Ur-perfomance, zu finden im Menü Works/Perfomance. Andere Aufführungen werden mit Performance getaggt.

Seitenangaben, Bandnummern u.ä.: Zeichnen Sie Seitenangaben bzw. Seitenumfänge, Bandnummern etc. mit der entsprechenden Bezeichnung aus dem Feld Bibilographic scope in dem Menü Bibliography aus.

Web adress : Web-Adressen (URLs) werden mit dem Tag web adress ausgezeichnet

DOI : DOI wird mit DOI ausgezeichnet

Opus/Katalognummern Zeichnen Sie Opuszahlen und andere Katalognummern mit Catalog number aus. Achten Sie darauf, sowohl die Abkürzung (»op.«, »KV«) als auch die Ziffer auszuzeichnen.

Widmung : Eine Widmung oder ein/e Widmungsträger/In wird mit Dedication ausgezeichnet.

Instrumentierung : Instrumentierung wird mit Instrumentation ausgezeichnet. Die vollständige Instrumentierung eines bestimmten Werkes wird mit einem einzigen Tag Instrumentation erfasst.

Tonarten : Eine Tonart in einem Werkverzeichnis wird mit Musical key ausgezeichnet.

RISM-Siglen : Hinweise auf RISM können mit einem von zwei Tags ausgezeichnet werden: RISM Series B/I und RISM library siglum. Für Series B/I setzen sie einen Schrägstrich vor die Nummer des Titels/Eintrags. Er erscheint in den veröffentlichten Texten automatisch als hochgestellte Ziffer. Beispiel: RISM 1610/16 erscheint in der Datenbank, d.h. in dem veröffentlichten Text als RISM 161016 . Die Tags sind zu finden in dem Menu Works.



UMI-Nummern : Für Dissertationen aus den USA kann eine UMI-Nummer mit UMI number ausgezeichnet werden, zu finden im Menu Bibliography.

Titel (Title)

Egal ob sie in einem Fließtextabschnitt, in einem Werkverzeichnis oder einer Bibliographie erscheinen, sollten Titel mit einem Tag aus dem Title-Menü ausgezeichnet werden. Wählen sie die entsprechende Option und der Text wird korrekt formatiert. Der Tag Title wird automatisch Kursivschrift erzeugen.

Titel innerhalb anderer Titel sollten auch auf diese Weise ausgezeichnet werden. Die Formatierung erfolgt automatisch, d.h. Titel innerhalb anderer Titel erhalten automatisch einfache Anführungszeichen.

Formatieren (Formatting)

Das Menü Formatting enthält verschiedene Tags für Textelemente, die entweder kursiv gesetzt werden oder in doppelten oder einfachen Anführungsstrichen stehen sollen.

Musical note name tagged eine Tonhöhe und setzt sie kursiv.

Pitch name zeichnet einen Notennamen aus mit Angabe der Oktave wie z.B. E2 or cis1. Die Ziffern müssen innerhalb des Tags {Pitch name} nicht separat ausgezeichnet werden, da sie automatisch formatiert werden.

Mentioned , So-called, and Foreign text kennzeichnet inhaltlich, was gemeint ist und sollte statt Italic oder Quotation marks verwendet werden.

Wenn Sie einen Begriff oder eine Wortfolge mit dem Tag Foreign Text auszeichnen, werden Sie aufgefordert, die Sprache des Textes einzugeben. Dieser Begriff oder die Wortfolge werden dann automatisch kursiv formatiert. Wenn der Begriff oder die Wortfolge nicht kursiviert werden soll, müssen Sie darüber hinaus ein Merkmal (render) eingeben.

Note: Anmerkungen (zu Überschriften) oder eckige Klammern in Zitaten. Der tag Note erfasst Textbausteine, die anders als der sie umgebende Text ausgezeichnet werden sollen. Es wird in den folgenden Fällen verwendet: Nach einer Überschrift zum Literatur- oder Werkverzeichnis erscheint oftmals eine Anmerkung, welche nähere Angaben zum Verzeichnis oder zu den dort gelisteten Quellen macht. Diese Anmerkung sollte als Note innerhalb des tags Heading ausgezeichnet werden.

Das Insert-Menü (Einfügen von Textelementen)

Mit dem Insert-Menü kann man Abschnitte, Sonderzeichen, Notenbeispiele, Abbildungen, Tabellen u.ä. in den Text einfügen. Vorhanden sind dafür die Menüs: Structure, List, Figure, Music example, Image, Link, Musical symbol, Special character, Gap, Editorial note und XML.

Abschnitte einfügen

Mit dem Menü Structure kann man Abschnitte oder Kapitel in einen Artikel einfügen bzw. diese nach ihrer Funktion auszeichnen. Das System unterscheidet nicht zwischen Kapitel und Unterkapitel. (vgl. allgemein zum Einfügen von Kapiteln und Abschnitten § A 4)

Heading section: Jeder Artikel beginnt mit einer Heading section (= Vorspann/Kopfteil, welcher beinhaltet: Lemma + ggf. Inhaltsverzeichnis bzw. Familienverzeichnis). Wenn es in dem Artikel um eine Person oder eine Familie geht, sollte das Lemma zusätzlich mit Personal name ausgezeichnet werden und dann mit seinen Gliedern (Surname, Forename usw).

Content section: Gibt es ein Inhaltsverzeichnis (oder eine Liste mit Familienmitgliedern), sollte dieser Abschnitt als Content section innerhalb der {Heading section} gekennzeichnet werden. Für Listen von Familienmitgliedern enthält der Content-Abschnitt ein Section Heading (für "Familie":) und einen Paragraph Paragraph für die Namensliste. Diese Namen brauchen in diesem Abschnitt nicht zusätzlich ausgezeichnet zu werden.

Zählung: Um die Zählung eines Kapitels zu ändern oder hinzuzufügen, muss man das number-Feld im properties-Menü benutzen. Man klickt die Überschrift an, geht dann in die Fußleiste auf den ( letzten) Button Section. Dann muss im number-Feld des properties-Menüs die Ziffer entsprechend geändert werden.

Tabellen können in FontoXML z.Zt. noch nicht erzeugt oder bearbeitet werden. Der in Tabellenzellen enthaltenen Text hingegen lässt sich bearbeiten, indem man den Kursor in eine Zelle setzt.

Autor/innen

Ein Autor (Document author) kann am Ende jedes Kapitels oder am Ende des Artikels eingefügt werden. Fügen Sie zunächst Author section ein und dort ein Document author. Autor/innen von MGG-Artikeln werden mit Personal name innerhalb von Document author ausgezeichnet.

Listen (List)

Um eine Liste zu erstellen, beginnen Sie einen neuen Absatz, dann klicken Sie auf den List-Button. Drücken Sie Enter, um eine neue Zeile innerhalb der Liste zu erstellen. (damit sind gemeint: Literaturverzeichnisse und Werklisten?)

Um die Hierarchie einer Zeile innerhalb der Liste zu verändern, müssen Sie tab oder Shift+ Tab drücken. Ersteres verschiebt den Eintrag bzw. die Zeile nach rechts, letzteres verschiebt ihn nach links. (das betrifft doch nur Inhaltsverzeichnisse?)

Inhaltsverzeichnisse

Um ein Inhaltsverzeichnis zu erstellen, müssen Sie folgende Schritte machen:

- Fügen Sie eine Content section innerhalb der Heading section ein,

- Dann fügen Sie eine Section heading hinzu und schreiben »Inhalt:« hinein

- Sie erstellen eine Liste mit list,

- Ersetzen Sie die Aufzählungszeichen (Punkte) durch Ziffern, indem Sie die Ziffer in dem number-Feld im Properties-Menü eingeben

- Die Ziffer muss für jede folgende Zeile korrigiert bzw. ergänzt werden.

Abbildungen (Figure)

Um ein Bild einzufügen, klicken Sie auf den figure-Button oder auf den Image-Button. Browsen Sie dann nach dem Bild oder der Abbildungen und klicken Sie auf upload. Wenn der Dateiname im Kasten erscheint, markieren Sie ihn und klicken auf select file. Sie können dann eine Legende (Überschrift und Text) schreiben.

Notenbeispiele (Musical example)

Ein Notenbeispiel Music example entspricht einer Abbildung Figure, außer dass es einen Ausschnitt notierter Musik enthält. Daher wird es auf dieselbe Weise wie eine Abbildung eingefügt (siehe 2.3.; anstatt auf den figure-Button klickt man hier auf den musical example-Button).

Abbildungen innerhalb von Textzeilen (Image)

Mit dem Button Image fügen Sie eine Abbildung innerhalb einer Textzeile ein. Es sollte benutzt werden, wenn die Abbildungen nicht mit Special character oder Musical symbol eingefügt werden können (d.h. dort nicht vorhanden sind).

Verweise (Link)

Hyperlink

Mit dem Tag Hyperlink können Sie einen Link zu einer Web URL einbauen. Schreiben Sie die vollständige URL in den Text. Dann markieren Sie die URL und Klicken auf den Tab Link/Hyperlink. Eine Box erscheint, in der Sie den URL erneut eingeben müssen.

Local link

Local links sind Links zu anderen Stellen im Artikel. Sie werden verwendet, um Links vom Fließtext zu den entsprechenden Stellen in den Werk- oder Literaturverzeichnissen zu erstellen.

Wenn Sie auf local link klicken, erscheint ein Screen, in dem Sie einen Ort eingeben können, zu dem Sie verlinken wollen. Beispiel: Ein Zitat, gefolgt von einem Verweis, wie z.B. (T. Guiard 1906): Markieren Sie diesen ganzen Verweis, klicken Sie Item in this document und suchen Sie »T. Guiard, Conferencia dada por Don Teófilo Guiard en la Salón de la Sociedad« im Literaturteil. Wählen Sie diese Passage aus und klicken Sie auf Insert.

Wurde ein Artikel geteilt und möchten Sie auf einen anderen Teil verlinken, müssen Sie Cross link wählen.

Cross link

Cross Links verweisen von einem Artikel auf einen anderen (oder im Falle einer Teilung eines Artikels von einem Teil auf einen anderen Teil). Sie werden verwendet, um Verweise zu verwandten Artikeln zu erstellen oder zu verwandtem Material innerhalb eines anderen Artikels.

Wenn Sie auf cross link klicken, erscheint ein Screen, in dem Sie den entsprechenden Artikelnamen oder einen anderen Ort eingeben können. Sie erhalten eine Vorschlagsliste von Artikeln, aus der Sie einen auswählen müssen. Sie klicken auf das Artikel-Icon und daraufhin insert; der Link wird gesetzt.

Wollen Sie nur auf einen bestimmten Teil eines Artikels verlinken, müssen Sie zunächst auf das Icon des entsprechenden Artikels klicken, dann zu dem entsprechenden Abschnitt scrollen und auf diesen Abschnitt klicken. Die gelb markierte Passage wird verlinkt.

Musikalische Sonderzeichen (Musical symbols)

Mit Musical symbols können Sie jedes von beinahe 2400 musikalischen Zeichen einsetzen. Sie sind nach »range« (Bereich) angeordnet, und die Bereiche können entweder numerisch sortiert werden (so dass häufiger verwendete Bereich immer oben erscheinen) oder aber alphabetisch. Diese Symbole sind dem Standard Music Font Layout (SMuFL) entnommen. In dem Feld »Filter lables and characters« kann nach einem Sonderzeichen oder nach einem Buchstaben gesucht werden.

Sonderzeichen (Special character)

Mit dem Menü Special character können nicht-musikalische Sonderzeichen eingefügt werden. Das tool ist ähnlich wie Insert symbol in Microsoft Word oder die Sonderzeichentabelle in Windows aufgebaut. Sie sind von Unicode range organisiert, und können entweder numerisch sortiert werden (sodass häufiger verwendete Zeichen immer oben erscheinen) oder aber alphabetisch. Man kann auch in dem Feld »Filter labels and characters« nach einem Sonderzeichen suchen. Ebenso kann ein Sonderzeichen auch aus einem anderen System wie z.B. Microsoft Word in FontoXML hineinkopiert werden.

Redaktionelle Anmerkungen (Editorial Note)

Mit Editoral note können Autoren, Herausgeber und andere Benutzer Kommentare in den Text einfügen. Sie sind im FontoXML gelb markiert und erscheinen nicht in der veröffentlichten Fassung, auch nicht wenn sie aus Versehen im Text stehen gelassen wurden. Zeichnen Sie einfach den Text als Editorial note aus oder fügen Sie einen Editorial note ein, setzten Sie den Cursor hinein und schreiben Sie Ihren Text.

XML einfügen (Text aus Word kopieren und einfügen)

Diese Funktion ist noch in Entwicklung, doch haben wir dafür eine provisorische Lösung erstellt. Die Funktion wird bald einfacher zu benutzen sein, doch im Moment lässt sich hiermit arbeiten.

Um etwas aus Word einzufügen, bearbeiten Sie das .doc- oder .docx-Dokument folgendermaßen:

  1. Nehmen Sie sämtliche Überschriften, Tabellen und Listen heraus, so dass das Word-Dokument nur aus Absätzen oder nicht nummerierten Listen besteht.
  2. Speichern Sie das Dokument.
  3. Gehen Sie zu http://mgg-edit.rilm.org/word-conv/ in einem Web-Browser
  4. Wählen Sie Choose file, dann finden Sie den word file , den Sie konvertieren wollen (.doc oder .docx)
  5. Öffnen Sie den file mit Notepad (auf einem PC), mit TextEdit (auf einem Mac) oder einem anderen Text Editor. (Fügen Sie keine Zeilensprünge oder Leerzeichen zu dem Dokument hinzu).
  6. Abschnitte: Kopieren Sie den Text von »<p>« bis »</p>«
  7. Tabellen: Kopieren Sie Text von »<item>« bis »</item>«
  8. Wählen Sie XML im FontoXML INSERT-Menü
  9. Fügen Sie Ihren Text mit Ctrl + V ein und klicken Apply

Das Table-Menü (Tabellen erstellen und bearbeiten)

Tabellen

Tabellen haben in diesem System weder eine Legende noch eine Zählung. Wenn Sie eine Legende oder eine Zählung benötigen, verwenden Sie Table figure (vgl. § 3.2 oben). Um eine Tabelle einzufügen, verwenden Sie den Table- Button im Table-Menü und wählen Sie die Anzahl der Zeilen und Spalten aus. Sie können jederzeit Spalten oder Zeilen hinzufügen oder löschen (mit dem Row- und Column-Menü).

Tabellen-Abbildung

Eine Tabellen-Abbildung ist in diesem System eine Tabelle mit einer Legende oder einer Überschrift. Das Feld »Überschrift« ist z.B. für eine Überschrift wie »Tabelle 3« gedacht. Die »Caption«/»Legende« dient der Beschreibung der Tabelle.

Das View-Menü (Ansicht)

Abschnittshierarchie (Section hierarchy)

Wenn Sie Section hierarchy aktivieren, wird die Hierarchie der Abschnitte mit optischen Balken und Einrückung auf der linken Seite angezeigt. Dies ist vor allem für längere Artikel mit komplexem Aufbau nützlich.

Leerschritte (Whitespace)

Wenn Sie Whitespace aktivieren, erscheint ein Punkt (∙) in jedem Leerschritt, d.h. jeder Leerschritt wird durch einen Punkt markiert.

Quelltext (Source)

Wenn Sie auf Source gehen, erscheint der zugrundeliegende XML-Code des Artikels.

Das Review-Menü

Rechtschreibprüfung (Spell check)

Wenn Sie Spell check aktivieren, werden alle Wörter, die sich nicht im Rechtsschreibewörterbuch von FontoXML befinden, mit einer roten Schlangenlinie unterstrichen. Klicken Sie mit der rechten Maustaste auf das unterstrichene Wort, um Rechtschreibvorschläge zu erhalten.

Änderungen verfolgen (Show changes)

Wenn Sie Show changes aktivieren, wird sowohl sämtlich hinzugefügter Text sichtbar (grün unterstrichen) als auch sämtlich gestrichener Text (rot unterstrichen) seit der letzten veröffentlichten Version des Artikels. Wenn Sie den Cursor an einer beliebigen Stelle des unterstrichenen Textes platzieren, können Sie sehen, wann und von wem die Änderung gemacht wurde. Änderungen können nur von der Redaktion angenommen oder verworfen werden.

Kommentare anzeigen (Show comments)

Wenn Sie Show comments aktivieren, sehen Sie sämtliche Kommentare, die seit der letzten Veröffentlichung des Artikels geschrieben wurden. Das Vorhandensein eines Kommentars wird durch die pinkfarbene Unterstreichung des Textes angezeigt. Setzen Sie den Cursor an eine beliebige Stelle dieses Textes, um den Kommentar zu sehen.

Kommentare hinzufügen (Add comment)

Mit dem Menü Comment können Autoren, Redakteure und andere Nutzer Kommentare in den Text einbauen. Um einen Kommentar zu hinterlegen, muss der Text unterlegt und dann Add comment geklickt werden. Es erscheint ein Feld, in das der Kommentar hineingeschrieben werden kann. Wenn Sie einen Kommentar hinterlegen, werden automatisch sämtliche Kommentare sichtbar (unterstrichen in pink). Um die Inhalte, den Verfasser und den Zeitpunkt des Kommentars zu sehen, muss der Cursor auf einer beliebigen Zeile des unterstrichenen Textes platziert werden. Um die pinkfarbene Unterstreichung zu löschen, muss der Tab Show comments im Menü View angeklickt werden.
