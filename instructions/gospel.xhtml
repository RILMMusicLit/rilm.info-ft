Encyclopedia of American Gospel Music (eag)
  1. This publication should be converted from the provided XML.
  2. Convert the contents of all <sc> elements to lower-case letters, and then remove the <sc> tags.
  3. Use American rule for single quotation marks.
  4. The tags <xref ref-type="page"/> should be converted to the TEI element <pb> with @n containing the page number:
    <xref ref-type="page" id="page_2"/>

    <pb n="2"/>
  5. The List of Contributors beginning on page ix should be set as a <list>. Each item may be tagged with <rilmPerson>, <persName>, and <orgName> or <placeName> as follows:
    <bio>
    <p><bold>Melvin L. Butler</bold> Dartmouth College</p>
    </bio>
    <bio>
    <p><bold>Richard Carlin</bold> New York, New York</p>
    </bio>

    <item>
    <rilmPerson><persName><forename>Melvin</forename> <forename><abbr>L.</abbr></forename> <surname>Butler</surname> <orgName type="uni">Dartmouth College</orgName></persName></rilmPerson>
    </item>
    <item>
    <rilmPerson><persName><forename>Richard</forename> <surname>Carlin</surname> <placeName>New York, New York</placeName></persName></rilmPerson>
    </item>
    Use the part of the name after the last space character as the <surname>. Exception: for names ending ", Jr." or "III", use <genName> for the part after the last space, and <surname> for the part preceding, as follows:
    <rilmPerson><persName><forename>Emmett </forename> <forename><abbr>G.</abbr></forename> <surname>Price</surname> <genName>III</genName>
  6. The List of Entries beginning on page xi can be set as a single <list> within a <div type="ref">:
    <list-item>
    <p>Abernathy, Lee Roy</p>
    </list-item>

    <item>Abernathy, Lee Roy L.</item>
  7. The table at the bottom of this page will be your guide to the value of @type in the <text> elements.
  8. All articles have the following sections: @head, @body, and at least one @ography. Many articles will have additional @ography sections, and some will have a <div type="ref"> preceding the @ography sections

    a) Inside the <div type="heading"> should be a <rilmPerson> for @type="person" articles, and a <p> for all others. Names before the comma should be placed in a <surname>, and names after the comma should each go in a <forename> element.

    Exceptions: the word "Reverend" should be enclosed in a <roleName> element; Anything in quotation marks should be put in <q> tags within an <addName> element; "Jr." or "III" should be put in a <genName> element.
    <title>Moore, Reverend Arnold Dwight “Gatemouth”</title>

    <div type="heading"><rilmPerson xml:id="r1"><persName><surname>Moore</surname>, <roleName>Reverend</roleName> <forename>Arnold</forename> <forename>Dwight</forename> <addName><q>Gatemouth</q></addName></persName></rilmPerson></div>

    b) <text type="person"> articles include birth and death dates and places in parentheses in the <p> element immediately following the article title. Sometimes another version of the person's name is included. All of these should be included in a <rilmPerson> element inside a <div type="subheading"> with an @corresp attribute pointing to the xml:id of the <rilmPerson> element above it. Tag the birth and death dates, and omit the parentheses and "b." and "d." The different name version should be set in a <persName> element.
    <title>Ford, Tennessee Ernie</title> <p>Ernest Jennings Ford (b. February 19, 1919, Ford-town, TN; d. October 17, 1991, Reston, VA)</p>


    <div type="heading">
    <rilmPerson xml:id="r1">
    <persName><surname>Ford</surname>, <forename>Tennessee</forename> <forename>Ernie</forename></persName></rilmPerson></div>

    <div type="subheading">
    <rilmPerson xml:id="r2" corresp="#r1"> <persName><forename>Ernest</forename> <forename>Jennings</forename> <surname>Ford</surname></persName>
    <birth><date when="1919-02-19" when-custom="1919">February 19, 1919</date>, <placeName>Fordtown, TN</placeName></birth>; <death><date when="1991-10-17" when-custom="1991">October 17, 1991</date>, <placeName>Reston, VA</placeName></death></rilmPerson>
    </div>
    c) Many <text type="group"> articles give a <list> of the group members within the first <p> following the <title> element. Put the <list> inside a <div type="subheading"> and tag each name as a <rilmPerson>:
    <title>Brown’s Ferry Four</title> <p><list list-type="simple">
    <list-item> <p>Alton Delmore (b. December 25, 1908; d. June 9, 1969)</p> </list-item>
    <list-item> <p>Rabon Delmore (b. December 3, 1916; d. December 4, 1952)</p> </list-item>
    <list-item> <p>Merle Travis (b. November 29, 1917; d. October 20, 1983)</p> </list-item>
    <list-item> <p>Louis Marshall “Grandpa” Jones (b. October 20, 1913; d. February 19, 1998)</p> </list-item> </list></p>

    <div type="heading"> <p>Brown’s Ferry Four</p> </div>
    <div type="subheading"> <list>
    <item><rilmPerson xml:id="p3"><persName><forename>Alton</forename> <surname>Delmore</surname></persName> <birth><date when="1908-12-25" when-custom="1908">December 25, 1908</date></birth>; <death><date when="1969-06-09" when-custom="1969">June 9, 1969</date></death></rilmPerson></item>
    <item><rilmPerson xml:id="p4"><persName><forename>Rabon</forename> <surname>Delmore</surname></persName> <birth><date when="1916-12-03" when-custom="1916">December 3, 1916</date></birth>; <death><date when="1952-12-04" when-custom="1952">December 4, 1952</date></death></rilmPerson></item>
    <item><rilmPerson xml:id="p5"><persName><forename>Merle</forename> <surname>Travis</surname></persName> <birth><date when="1917-11-29" when-custom="1917">November 29, 1917</date></birth>; <death><date when="1983-10-20" when-custom="1983">October 20, 1983</date></death></rilmPerson></item>
    <item><rilmPerson xml:id="p6"><persName><forename>Louis</forename> <forename>Marshall</forename> <addName><q>Grandpa</q></addName> <surname>Jones</surname></persName> <birth><date when="1913-10-20" when-custom="1913">October 20, 1913</date></birth>; <death><date when="1998-02-19" when-custom="1998">February 19, 1998</date></death></rilmPerson></item></list></div>
  9. Sections within an article--<sec> elements in the source xml beginning with a <title>--should begin a new <div> each with a <head> element.
    <sec id="S51"> <title>Origins</title> <p>The origins of barbershop harmony...</p></sec>
    <div type="section"> <head>Origins</head> <p>The origins of barbershop harmony...</p></div>
  10. Many articles include quotations from song lyrics enclosed in a <verse-group> and <verse-line> elements. These should be placed within the preceding <p> element (if present, otherwise in its own), in <lg> tags, with each line in its own <l> tag:
    <verse-group>
    <verse-line>Sweet Adeline (sweet Adeline)</verse-line>
    <verse-line>My Adeline (my Adeline)</verse-line>
    <verse-line>At night, dear heart (at night, dear heart)</verse-line>
    <verse-line>For you I pine (for you I pine)</verse-line>
    </verse-group>

    <lg>
    <l>Sweet Adeline (sweet Adeline)</l>
    <l>My Adeline (my Adeline)</l>
    <l>At night, dear heart (at night, dear heart)</l>
    <l>For you I pine (for you I pine)</l>
    </lg>
  11. Author names come at the end of the body text in a <disp-quote> element, in forename-surname order. Place in a <div type="authors"> using the part after the last space character as the surname; IF the name ends with ", Jr." or "III", then set the last part in a <genName> element:
    <disp-quote><attrib>CHARLES K. WOLFE</attrib></disp-quote>

    <div type="authors"><docAuthor><persName><forename>CHARLES</forename> <forename><abbr>K.</abbr></forename> <surname>WOLFE</surname></persName></docAuthor></div>
  12. See also's: Paragraphs in boldface that begin "See also" should be set as a <list> within a <div type="ref">. References are separated by semicolons; each points to another article in this publication, and will have a <ref> element with @target pointing to the appropriate article:
    <p><bold>See also Hall, Connor; Homeland Harmony Quartet; Rangers Quartet</bold></p>

    <div type="ref">
    <p>See also </p><list><item><ref target="eag000??">Hall, Connor</ref></item> <item><ref target="eag000??">Homeland Harmony Quartet</ref></item> <item><ref target="eag000??">Rangers Quartet</ref></item></list>
    </div>
  13. Bibliographies are titled “Reference and Further Reading". Discographies may be titled “Discography", “Selected discography", or “Compilations".

    a) Some discographies and "selected recordings" lists are inline, where items are separated by semi-colons; some have each item on its own line. In the TEI-XML, you do not need to differentiate between these. All will be <list> with <item> children.

    b) Some Discographies have subsections, such as <sec id="S25"> and <sec id="S26"> in the source XML. Each can be a separate “ography".
  14. If we decide to tag the terms in the Index at the end of the book as described in the main instructions, then the list itself does not need to be converted.

Please refer also to the two provided xml samples.
