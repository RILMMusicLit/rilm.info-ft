Real-Lexikon der Musikinstruments (Sachs)

This title will be converted from scanned print. Please create a whole-volume PDF as well as article-level TEI and METS-ALTO.

All entries will be <text type="instrument">.

Many entries include non-Latin script—Arabic/Persian, Cyrillic, Greek, Hebrew, Sanskrit. If the non-Latin text cannot be capture via OCR, replace it with a <gap> element with an @xml:lang attribute giving the language of the omitted text. (see sachs-language-codes.xhtml)

There is a lot of information given in the entries, including language, translations, synonyms, and plurals.

Quotation marks are of the German low-high type (‚ ‛); They have different specific meanings in this work, depending on context, and should not be substituted with "<q>". Their interpretation is described below.

  1. Front Matter:
    • Dust jacket notes (on inside folds; not the outside covers).
    • Advertisement page ("American Musicological..."; omit the half-title)
    • Title page
    • Copyright page
    • Publisher's Note
    • Introduction to Dover Edition; use <q> tags for the quotation marks in this section only.
    • Vorwort. Include the "Use of the dictionary" as a div type="section".
    • "Abgekürzt zitierte allgemeine..." (bibliographical abbreviations)
    • "Abgekürzt zitierte Instrumenten..." (instrument collections' abbreviations)
    • "Abkürzungen der häufigsten..." (language and ethnic abbreviations)
    • "Transkriptions-schlüssel"
  2. Dictionary entries titles are in bold, capitalized.
    • See-references appear in these forms:
      1. Term1 s. Term2. [e.g. Abhubh, s. Abub.]
      2. Term1 = Term2 [e.g. Arbeitsbalg = Schöpfbalg.]
      Do not make a TEI file for these two kinds of entries; instead, include the heading term in the <keywords> element of the target entry (refer to the general rules for see references).
      For all other types of entries, you will create a TEI file as below.
    • Capture the bold-type heading as the <title> in the <teiHeader>. Use the same text for the <div type="heading">, and tag it as <term>. Omit the comma following the heading in both elements.
    • There several patterns that entries might follow after the heading:
      1. Term1, lang. ‚Term2‛ [e.g. Agujero, span. ‚Griffloch‛.] Tag the heading as term with xml:lang if possible (see sachs-language-codes.xhtml); tag term2 (the term in single quotes) with <ref> pointing to the entry for that term.
        <div type="heading" xml:id="d1"><p xml:id="p1"><term xml:id="t1" xml:lang="es">Agujero</term></p></div>
        <div type="body" xml:id="d2"><p xml:id="p2">span. ‚<ref target="srl00000">Griffloch</ref>‛.</p></div>
        There may be more than one term within the quotation marks; tag each with <ref> if a matching entry is found [Cassa, ital. ‚Trommel, Gehäuse‛.].
        <div type="heading" xml:id="d1"><p xml:id="p1"><term xml:id="t1" xml:lang="it">Cassa</term></p></div>
        <div type="body" xml:id="d2"><p xml:id="p2">ital. ‚<ref target="srl00000">Trommel</ref>, Gehäuse‛.</p></div>
      2. Term1, Term1-original-script, lang. ‚Term2‛ [e.g. Čekü, چكو, pers. ‚Plektrum, Bogen, Schlägel‛.]
        <div type="heading" xml:id="d1"><p xml:id="p1"><term xml:id="t1" xml:lang="fa">Čekü</term></p></div>
        <div type="body" xml:id="d2"><p xml:id="p2"><term xml:id="t2" xml:lang="fa" corresp="t1">چكو</term> pers. ‚<ref target="srl00000">Plektrum</ref>, <ref target="srl00000">Bogen</ref>, Schlägel‛.</p></div>
      3. Term1, T e r m 1 v a r i a n t, lang. ‚Term2‛ [e.g. Crepitaculum, C r e p u n d i a, lat. .Klapper, Schelle'.
      4. Term1, plur. v. Term2 [e.g. Stuic, plur. v. Stoc.]
        <div type="heading" xml:id="d1"><p xml:id="p1"><term xml:id="t1">Stuic</term></p></div>
        <div type="body" xml:id="d2"><p xml:id="p2">plur. v. <ref target="srl00000">Stoc</ref>.</p></div>
      To summarize some typographical conventions:
      1. Text within quotation marks indicates a reference only when it immediately follows a language abbreviation. Later in the entry, quotation marks are used for quoting. See example file 1 for the distinction.
      2. L e t t e r s p a c e d text marks a variant of the preceding term.
  3. Small type at the ends of entries may contain bibliographical references (either abbreviated or written-out), keywords, or references. Since there is no reliable way to identify which it is, tag as <note> as shown in the example files.
  4. Name index. Please search the listed lames on the listed pages and tag <persName>.
