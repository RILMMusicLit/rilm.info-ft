Guidelines for RILM Journals Collection PDFs

  1. PDF Version
    PDFs should be saved as PDF version 1.6. PDFs we receive in PDF version 1.7 (and above) should be converted to version 1.6.
  2. Tagged PDF
    PDFs should be tagged.
    • for <figure> use @alt or @actualText attribute when possible
    • use <table> element only for actual tables, not for layout
    • tables should have headers
    • Heading elements should be properly nested (<h2> within <h1>--do not skip levels)
    • Heading elements should not have descendents
    • The list element type <L> requires a @ListNumbering attribute which designates the numbering system used in the list, e.g. Disc for a simple bullet without numbers, Decimal, or UpperRoman.
  3. Copyright Page
    A copyright page should appear at the end of each document.
  4. Unlocked for Editing
    PDFs must be unlocked to allow for editing.
  5. Searchable Text
    Except for exceptions explicitly authorized by RILM, PDFs must have an embedded searchable text layer.
  6. Deskewing
    Scanned documents should not be skewed more that one degree.
  7. File Size
    PDF files cannot be larger than 1GB. For files over 300KB per page, file size should be reduced if it can be done without compromising text or image quality. For files over 600KB per page, file size should be reduced if it can be done without a significant loss of image or text quality.
  8. Embedded Fonts
    Fonts should be embedded when possible and not encrypted.
  9. Color Profiles
    PDFs should have an ICC profile.
  10. PDF Metadata
    The following metadata fields are desired:
    • xmp:CreatorTool [=creator (DocInfo)]
    • xmp:CreateDate
    • xmp:MetadataDate
    • xmp:ModifyDate [=ModDate (DocInfo)]
    • dc:title [=Title (DocInfo)]
    • dc:format
    • pdf:PDFVersion
    • pdf:Producer [=Producer (DocInfo)]
