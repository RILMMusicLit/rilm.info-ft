Enciclopédia da música brasileira: erudita, folclórica, popular
  1. General comments:
    • This publication should be converted from provided PDFs.
    • Articles include topics, instruments, persons, and groups.
    • Tag italic type as <title> if it begins with an capital letter; <hi rend="italic"> if ti begins with a lower-case letter.
    • High quotation marks (“ ”) are used.
    • Dates are given as dd/mm/yyyy throughout.
  2. Front matter (Abertura.pdf)

    • Copyright page
    • Editors lists (Primeira, Segunda, Terceira; pp. 3-5)--make three <div type="section">, one for each page. Tag each bold-capital as <head>, what is in-between as <list>.
    • "Tábua de matérias" (Table of contents).
    • "Prefácio" (Preface)
    • "Notas para uso da enciclopédia" (Note for users; pp. 9-11)--like the editors lists, make three <div type="sections> (Primeira, Segunda, Terceira), and ordered lists in between.
    • "Abreviaturas" (abbreviations; pp. 12-19): tag as a two-column <table>.
    • OMIT the repeated title page (p. 21).
  3. Topic articles can be distinguished by having normal-case bold headings; usually they are all lower case. Use the text in bold for the <title> in the TEI header and for the <div type="heading">. Use the remaining text for <div type="body">.
    • If the text begins "O mesmo que", it is a see-reference. Don't make a file for it.
    • When a given term means two or more different things, the entries' headings are followed by superscript numerals. Retain these as they are.
  4. Instrument articles are the same format as topic articles, but the text begins "Instrumento".
  5. Person articles have headings that have either all-caps-bold followed immediately by parentheses, OR all-caps-bold followed by normal-case bold.
    • Where the heading is only all-caps, use the bold text for the <title> element in the TEI header and for the <div type="heading">. Use the part in parentheses as the beginning of the <div type="body">.
      • Tag the bold text as a whole as <rilmPerson><persName><addName type="nickname">.
      • Tag the name in parentheses as <rilmPerson> corresponding to the first. Assume that all the names in parentheses are <forename> except for the last, which is <surname>. Tag <nameLink> if you can identify as such.
      [CASQUINHA (Oto Henrique Trepte).]
      <div type="heading"><p><rilmPerson xml:id="r1"><persName><addName type="nickname">CASQUINHA</addName></persName></rilmPerson></p></div>
      <div type="body"> <p><rilmPerson xml:id="r2" corresp="r1">(<persName><forename>Oto</forename> <forename>Henrique</forename> <surname>Trepte</surname></persName>)....
    • Where the heading has all-caps-bold followed by normal-case bold, again use the bold text for the <title> element in the TEI header and for the <div type="heading">.
      • Tag the bold text as <rilmPerson><persName>, NOT with <addName>.
      [CASTRO Roberto de.]
      <div type="heading"><p><rilmPerson xml:id="r1"><persName><surname>CASTRO</surname> <forename>Roberto</forename> <nameLink>de</nameLink></persName></rilmPerson></p> </div>
      • IF there is a name in parentheses following, tag as described above.
      [CASTILHO Lindomar (Lindomar Cabral).]
      <div type="heading"><p><rilmPerson xml:id="r1"><persName><surname>CASTILHO</surname> <forename>Lindomar</forename></persName></rilmPerson></p></div>
      <div type="body"> <p><rilmPerson xml:id="r2" corresp="r1">(<persName><forename>Lindomar</forename> <surname>Cabral</surname></persName>)....
    • The body text of Person articles—beginning after the name in parentheses, if there is one—begins with occupations followed by birth and death information. INCLUDE these within the second <rilmPerson> element. See the example files.
    • Worklists in Person articles can be incomplete, called "Obras" ("works"), or complete, called "Obra completa" ("Complete work").
      • Some "Obras" lists have subheadings. Tag the subheadings <p>. Make a <list> after each one, See example file emb00002.
      • "Obra completa" lists have two levels of subheadings. Makea <div type="section"> for each subheading on its own line. See example file emb00005.
  6. Group articles have headings in all-caps-bold (no normal case), NOT followed by parentheses.
    • Use the bold text as <title> in the teiHeader and tag as <term> within the <div type="heading">.
      [14 BIS.]
      <div type="heading"><p><term>14 BIS</term></p></div>
  7. See-references have two different forms. Treat both kinds according to http://rilm.info/ft/tei_rilm_instructions.html#see-refs
    • [term O mesmo que target.] Only used for terms / topics. The abbreviation following the referring term is the part of speech, and can be ignored in the case of references like this.
    • [NAME name. Ver TARGET.] Used for all types of entries.
    • Cross references may appear at the ends of articles, in the form [Ver target.] References can also be inferred within the body text by the presence of superscript numerals. See example file emb00003 for both kinds.
