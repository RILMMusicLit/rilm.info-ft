Diccionario de la Música Española e Hispanoamericana (meh)

We are converting only a sample of this publication, which we will present in a project proposal to its editors and other stakeholders. Because these documents will be presented without a finished content delivery platform, we will put an unusual stress on their appearance with a dedicated stylesheet, which can be found at http://www.rilm.info/ft/css/tei_dmeh.css. In addition to your usual quality control processes, please view each XML document with this stylesheet and compare it to the PDF. Any differences should be reported to RILM.

This publication is to be converted from QuarkXpress files.

  1. <teiHeader>:
    • The <title> element within <titleStmt> should contain the boldface text at the start of each article.

    • Create a <change> element for each article created with an @when attribute and @status="created".

  2. See references: See references appear in very short entries with the keyword "veáse" in italics. Information in these entries will be copied to their target entry, as instructed here, but with two changes:
    • In addition to adding a <term> entry to the target article, create an file for the entry itself.
    • In the <term> element, the boldface headword will be followed by a Roman description, which will appear in a <gloss> element:


      <textClass>
      <keywords>
      <term>Bandullo<gloss>Venezuela. Membranófono de frotación</gloss></term>
      </keywords>
      </textClass>
      See references ("Véase") in longer articles do not need to be added to target entries as <term> elements.
    • In all cases that "véase" appears in italics, it should be stripped out and the references should be set in <div type="ref"> as follows. The smallcaps formatting of the references should not be retained.


      la pierna izquierda casi se arrastra transportándolo.</p> <div type="ref"><p><ref target="mehEXAMPLE3">Caja</ref>; <ref target="mehEXAMPLE4">Tambor</ref>; <ref target="mehEXAMPLE5">Tambora</ref>.</p></div>
  3. Text types: Please use text types listed in this chart for each article.
  4. <div> elements should receive one of the following @type values: "heading", "section", "one-section", "person", "works", "ography", or "authors".
  5. Here are three basic structures for <div>'s:
    1. Family articles (<text type="family">):
      <div type="heading"></div>
      <div type="person"></div>*
      <div type="person"></div>
      (<div type="person"></div>...)
      <div type="authors"></div>
    2. Articles without a table of contents (excluding family articles):
      <div type="one-section"></div>
      (<div type="works"></div>)
      (<div type="ography"></div>)
      <div type="authors"></div>
    3. Articles with a table of contents:
      <div type="heading"></div>
      <div type="contents"></div>
      <div type="section"></div>
      <div type="section"></div>
      (<div type="section"></div>...)
      <div type="authors"></div>
    *<div type="person"> section will contain all of the section that non-family articles contain. Thus the structure of Barbieri (the only family article in the current selection) will be as follows:
    <div type="heading"><head></head><p></p></div>
    <div type="person" n="1"><div type="one-section"><p n="1"><rilmPerson></rilmPerson></p></div></div>
    <div type="person" n="2">
    <div type="heading"><p n="2"><rilmPerson></rilmPerson></p></div>
    <div type="contents"><list></list></div>
    <div type="section" n="I"><p n="I"></p>...</div>
    <div type="section" n="II"><p n="II"></p>...</div>
    <div type="section" n="III"><p n="III"></p>...</div>
    <div type="section" n="IV"><p n="IV"></p>...</div>
    <div type="section" n="V"><p n="V"></p>...</div>
    <div type="section" n="VI"><p n="VI"></p>...</div>
    <div type="works"></div>
    <div type="works"></div>
    <div type="ography"></div>
    </div>
    <div type="authors"></div>
  6. For <text type="person"> articles, the @heading div will contain a <rilmPerson> element; for all other articles that require one, the @heading div will contain <head> for the boldface headword and <p> for the rest of the text.
  7. Body text:
    • Outside of an initial <rilmPerson> for <text type="person"> or <div type="person">, the body text should only contain formatting tags such as <hi>, <q>, <p>, and <list>/<item>.
  8. Figures:
    • Figures in DMEH are not numbered. All figures, including those containing musical notation, should be set in a <figure> element. The element will have an @xml:id, but not an @n attribute.
    • Captions will be set in <p>.
    • Rename images with the same convention used for MGG. The first image in meh00001 would be meh00001-f1.
  9. Works lists and bibliographies:
    • Sections with the following headings should be <div type="works">: OBRAS, ESCRITOS MUSICALES, OBRAS DIDÁCTICAS.
    • Sections with the following headings should be <div type="ography">: BIBLIOGRAFÍA, DISCOGRAFÍA.
    • <div type="ography"> sections are presented in the source as in-line lists with each item separated by a semicolon. That semicolon and any other space between items should be deleted, so that there are no spaces between the final letter of one item and the first letter of the next. Each list should be presented on one line of code.
    • The <bibl> within each item should only contain <author>, <editor>, <title>, <title rend="quoted">, and <date> (and if necessary <hi>).
    • Authors' names should only be tagged at the start of a <bibl>. Otherwise leave names untagged.
    • <editor> should only be used for names that directly follow "ed." in lower case.
    • Neither <author> or <editor> should have descendent tags. Do not use <forename>, <surname>, etc. within <author> or <editor>.
    • <title> should be used for all italicized titles in works lists and ographies.
    • <title rend="quoted"> should be used for all titles set in quotes in works lists and ographies.
    • <date> should be used for four-digit years within <bibl> (but not within titles).
