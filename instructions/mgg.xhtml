Die Musik in der Geschichte und Gegenwart (mgg)

This publication is to be converted from XHTML (.html) files to our TEI-XML format.

  1. ID and filename generation. The ID of each article will begin with the lowercase letters "mgg" followed by an auto-generated five-digit number. The filename for that XML file will consist of that ID followed by the .xml file extension.

  2. teiHeader:

    • Create a <change> element for each article created with an @when attribute and @status="created".

    • The <title> element within <titleStmt> should be the contents of <div type="heading> without any alternate spellings or names. Thus, while the heading of an article in the Personenteil might read "Gerbič, Gerbić, Gerbec, Fran, Franjo", The titleStmt/title contents would just be "Gerbič, Fran".

  3. Each <text> element (one per article) should have a @type attribute with one of the following values: "person", "family", "corporate", "topic". It should also have @xml:id with the article ID as value, and @xml:lang="de-DE-1901".

    Articles in the Personenteil that are about neither a person nor a family should be <text type="corporate">. These articles do not begin with birth information (the * character). For these articles,text in the <div type="heading"> should be an <orgName>, not a <persName>.

  4. <div> elements should receive one of the following @type values: "heading", "contents", (when within <front>); "section", "one-section", "bio", "criticism", "works", "ography", or "authors" (when within <body>). In person articles in the Personenteil, one works section will always appear after the biographical section, and it will usually the heading "Werke". (Any list that appears before the second prose section should be type="works".) All other lists (which appear after a second prose section) will have various headings, but will all receive the @type value "ography". While there is generally only one "works" section (often divided into sub-sections), there can be many "ography" sections. The section between these two sets of lists does not have a heading. It should have @type="criticism". If there are named sections within any of the top-level sections (bio, works, criticism, ography), they will receive an @type="section" attribute. All lists in the Sachteil should have the @type value "ography". If a Sachteil article is divided into sections, each should have a <div type="section">. If there is only one section, it should have <div type="one-section">. The same is true for articles in the Personenteil that are not about persons.

  5. Family entries: there are many entries for families which contain sub-entries for family members. The <text> should have @type=”family”. Also, in these cases, a <div> should appear around the each family member’s section. These articles begin with a list of the family members. This list should be contained in a <p>, and should appear within a div type="contents". The word “Familie” should appear in a <head> element in this div.

    Example

    <div type="heading" xml:id="d1">

    <p xml:id="p1">Spohr</p>

    <div type="contents" xml:id="d2"> <head>Familie:</head><p xml:id="p2">Louis (1), sein Bruder Ferdinand, seine erste Frau Dorette...</p></div>

    </div>

    Within family articles each <div type="person"> begins with forenames folowed by biographical information. All of this should be tagged with <rilmPerson>. All of the names here are forenames.

    <div type="person"> sections can contain all div types that <text type="person"> can contain. If a a works list an ography, or a criticism section appear before the next family member, those sections should always be placed inside the <div type="person">. The following are two examples of structures for this type of section:

    <div type="person">
    <div type="bio"></div>
    <div type="works"></div>
    <div type="criticism"></div>
    <div type="ography"></div>
    </div>
    <div type="person>
    <div type="bio"></div>
    <div type="ography></div>
    </div>

  6. Author’s Names: Name of article authors are to be tagged in <persName> elements within a <docAuthor> element. Parts of the name should be tagged with <forename>, <surname>, <nameLink>, <genName>, <roleName>, and <addName>. This <docAuthor> element goes in a <div> with @type="authors". If there are multiple authors for an article, enclose each <persName> elements within a separate <docAuthor>. Be certain that the <docAuthor> for the article is contained within the <div> for the article, but not within any child <div>. If the authorship of an article is broken down into sections, the <div> for each section should contain a <docAuthor> element, in addition to the one for the whole article.

    Here is how to determine authorship for individual secitons: a name at the end of one section indicates authorship of all preceding sections that do not include an author’s name. For instance, if an article contains five sections and Peter Swanson wrote sections 1,2, and 3, and Jane Weiss wrote sections 4 and 5, the name “Peter Swanson” would appear only after section 3, and the name “Jane Weiss” would appear only after section 5.

  7. Quotation marks: MGG uses guillemets (» and «; › and ‹ ) rather than English-style quotation marks (“ and ”; ‘ and ’). All guillemets (but not apostrophes) should be replaced with TEI elements. When a more meaningful element cannot be determined, these characters can always be redered with the <q> element (with @rend="single" for single guillemets). However, use <quote> or <title> when they can be applied with confidence. Here are some guidelines for those elements:

    • <title> should be used in bibliographies, works lists, and the like for items phrases in italics--both in guillemets and without guillemets. Most items in these lists will have a <title> element.

    • <quote> should only be applied to quotations within the text that both are italicized and have a reference to a source. In MGG, these references are in parentheses, but they do not always appear directly after the quote. <quote> elements will appear either (a) along with a <ref> element within a <cit> element, or (b) on their own with an @source attribute.

    • Some quotations must be tagged with <quote> within <cit> because it can be used as a block-level element that contains paragraphs, lists, or tables.

  8. Personenteil article headings: Most headings in the Personenteil consist of a list of names separated by commas. The preferred surname is in boldface, Roman type; the preferred forename is is non-bold Roman type. The alternate names are in italics. When there are no alternate names, that is, there are only two names and one comma, that name can go into one <persName> element. If there are any variants, each name should go in a separate <persName> element (in that case, each <persName> will contain either a surname or a forename, but not both). All variants before the preferred forename should be assumed to be surnames and should have @type="alt" attributes. All variants after the preferred forename should be assumend to be forenames and should also have @type="alt" attributes.

    Some Personenteil headings also include abbreviations indicating nicknames, sobriequets, legal names etc. These include "gen." and "eigentl."

  9. Birth, death, and occupation data. This data appears at the start of the (prose) biographical section of each Personenteil entry. The paragraph (<p>) that contains this data should begin with a <rilmPerson> element. Within this element, elements such as <birth>, <death>, <baptism>, and <burial> should be inserted when appropriate. MGG uses a asterisk (*) to signify birth information and a dagger (†) for death information. The abbrevation for baptized is “get.” These markers should each be placed in an <abbr> element. The word for buried is “begraben”--no tagging is necessary for this word. A date goes in the <date> element with two attributes: @when attribute for the date in WC3 format; and an @when-custom attribute for a four-digit year. All dates within <rilmPerson> should have both attributes. When a date is preceded by the word “um”, an additional attribute should be added: @precision="low". A list of occupations common in MGG can be found here. Occupations found on that list can be tagged <occupation>

    When there is a range such as "1505-1514", please use the middle year in for both attributes and add @precision="low":

    <birth><date when="1510" when-custom="1510" precision="low">1505-1514</date></birth>

    For a date such as "26./27. Sept. 1614" choose the earlier date and add @precision="low".

    After the first sentence of an article, the <rilmPerson> element should always include a name (this is likely to require input from the Editorial Department). For example:

    Sein jüngerer Bruder <rilmPerson>Charles (<birth><date when="1511" when-custom="1511">1511</date></birth></rilmPerson>)

    When a <birth> or <death> element is added, the "*" and "†" should always be deleted.

  10. Notes and pitches. Note names and pitch names should be marked with <musNote> and <pitch>, respectively. Note names in German are Ces C Cis Des D Dis Es E F Fis Ges G Gis As A Ais B H. In MGG they are always rendered in italcs. Pitch names are those note names followed by a subscript or superscript number. So f2 would be rendered as <pitch>f2</pitch>

  11. Arrows before cross-references: The arrow and the following space that precede cross-references should be deleted.

  12. Bibliographic and works lists

    1. Superscript numbers indicate edition numbers (except for those that follow "RISM" and a year). These can go in <edition> elements.

    2. The exception to this is superscripts that follow "RISM" and a year. These should be tagged with <hi rend="sup">

    3. Anything is a <title> tag is by default italic. Titles that have other styling should have an @rend attribute to indicate whether the title was roman, quoted, or quoted-italic.

    4. "hrsg. von" ("edited by") indicates that the following person is an editor and should go in an <editor> element.

    5. If the name of an editor is rendered in smallcaps, please use the following tag: <editor subtype="primary">

    6. When an author or editor is an organization, use the <orgName> tag within <author> or <editor>. If it is unclear, do not tag anything within the <author> or <editor> tag.

    7. Cross-references: Cross references are indiated with an arrow in the XHTML files. Make a link to the appropriate article ID.

    8. If there are two titles and two dates of publication, there should probably be a secondary <bibl> within the primary <bibl>

    9. In Roman titles that follow "in:", any time a number follows the title, that number should be tagged with <biblScope>. The big exception to this is four-digit years, which should be tagged as part of the title. This only applies to Roman titles that follow "in:". Numbers followed by periods should not be tagged with <biblScope>

    10. Do not use <instrumentation> for MGG.

    11. Do not tag dates (or places)  within titles. If a date appears within a title, leave it untagged.

    12. Only tag dates within <bibl> when they are in the form dddd (where d is a digit).

    13. For list sections with headings other than those in the following list, use only the tags <item>, <bibl>, and <hi>: "Werke", "literatur", "ausgaben", "schriften", "vertonungen", "editionen", "textausgaben",  "briefausgaben".

    14. Do not use the <pubPlace> tag in works lists <div type="works">.

    15. Unless the heading (or sub-heading) is "Schriften", <div type="works"> lists should only receive the tags <title> (without attributes), <date>, and (if applicable) <editor>.

    16. In sections with the heading "diskographie": (1) Do not tag anything except for titles and names. Anything else that is italic should be <hi rend="italic">; (2) In these sections, words within parentheses are not titles. Please tag them simply with <hi rend="italic">; (3) Years within parentheses in these sections should simply be tagged <date> (not date type="comp")

    17. Many "works" or "ography" sections begin with some prose text, either in parentheses or not. Text that appears after the heading, indented on a new line, should be tagged with <p>. Text that appears after heading, either on the same line or on a new line but not indented, should be tagged in <note>.

    18. <title rend="roman"> only in the following cases:
      • after "in:" (must include the colon)
        <bibl xml:id="b8"><author><forename><abbr>J.</abbr></forename> <surname>Hodin</surname></author>, <title>Bedrich Smetana i Göteborg</title>, in: <title rend="roman">Ord och Bild</title> <date>1941</date>, <biblScope unit="pp">537-547</biblScope></bibl>
      • after "in: " plus the name of an editor or author (see NB below)
      • within parentheses that are roughly the same length as and _directly_ follow an italicized title (these are translations).
        <bibl xml:id="b17"><author><forename><abbr>A.</abbr></forename> <surname>Nordström</surname></author>, <title>Det hände sig i Göteborg</title> (Es geschah in Göteborg), <pubPlace>Göteborg</pubPlace> <date>1980</date></bibl>
      • around "dass."
        <bibl xml:id="b49"><author><forename><abbr>A.</abbr></forename> <surname>Zobel</surname></author>, <title>Die Kirchenbibliothek der Peterskirche zu Görlitz</title>, in: <title rend="roman">dass.</title>, <biblScope unit="vol">Bd. 117</biblScope>, <date>1941</date></bibl>

      N.B.: when "in:" is followed by a name (with "hrsg.") and then a comma, it should be tagged <editor> (if followed by "Hrsg.") or (and this is very rare) <author rend="roman">. That will be followed either by <title rend="roman"> or <date>.

      <bibl xml:id="b23"><author>Ders.</author>, <title>Jenaer Liederhs</title>., in: <editor><forename><abbr>K.</abbr></forename> <surname>Ruh</surname></editor> <editor>u.a.</editor> (Hrsg.), <title rend="roman">Die dt. Literatur des MA. Verfasserlexikon</title>, <biblScope unit="vol">Bd. 4</biblScope>, <pubPlace>Bln./N.Y.</pubPlace> <edition rend="sup">2</edition><date>1983</date>, <biblScope unit="col">Sp. 512-516</biblScope></bibl>
      <bibl xml:id="b7"><author><forename><abbr>Chr.</abbr></forename> <forename><abbr>G.</abbr></forename> <surname>Körner</surname></author>, <title>Über Charakterdarstellung in der Musik</title> (<date type="comp">1795</date>), in: <author rend="roman"><forename><abbr>W.</abbr></forename> <surname>Seifert</surname></author>, <title rend="roman">Christian Gottfried Körner. Ein Musikästhetiker der dt. Klassik</title>, <pubPlace>Rgsbg.</pubPlace> <date>1960</date>, <biblScope unit="pp">147–158</biblScope></bibl>
      <bibl xml:id="b21"><author><forename><abbr>P.</abbr></forename> <surname>Boulez</surname></author>, <title>Nahsicht und Fernsicht</title>, in: <author rend="roman">ders.</author>, <title>Werkstatt-Texte</title>, <pubPlace>Bln.</pubPlace>/<pubPlace>Ffm.</pubPlace>/<pubPlace>Wien</pubPlace> <date>1972</date>, <biblScope unit="pp">58–75</biblScope></bibl>

    19. <meeting> should start with "Kgr. Ber." (or "Kgr.Ber.") but continue until next comma. This will often include a year and a place abbreviation, but those should not be tagged as such.

      <bibl xml:id="b18"><title>Kolloquium: KM. im Spannungsfeld zwischen Kunst und Gottesdienst</title>, in: <meeting>Kgr.Ber. GfM Bayreuth 1981</meeting>, <pubPlace>Kassel</pubPlace> <date>1984</date>, <biblScope unit="pp">233-277</biblScope></bibl>
    20. Periods in <bibl> elements usually do not separate elements. They are usually serving as one of three things:
      • part of an abbreviation
      • part of an ordinal number ("8." in German = "8th" in English).
      • the division between a parts of a title: <title rend="roman">Christian Gottfried Körner. Ein Musikästhetiker der dt. Klassik</title>
      This means that they the only case where they would appear at the end of a title is as part of an abbreviation:
      <bibl xml:id="b8"><author><forename><abbr>A.</abbr></forename> <surname>Schubiger</surname></author>, <title>Die Sängerschule St. Gallens vom 8. bis zum 12. Jh.</title>, <pubPlace>Einsiedeln</pubPlace>/<pubPlace>N.Y.</pubPlace> <date>1858</date></bibl>
    21. In general, restrict the use of <pubPlace> and <date> to places matching the one of the following two patterns:
      • <pubPlace> <date> (or <pubPlace> <date>, <biblScope unit="pp">)
        <bibl xml:id="b6"><author><forename><abbr>J.</abbr></forename> <surname>Noppius</surname></author>, <title>Aacher chronick</title>, <pubPlace>K.</pubPlace> <date>1632</date></bibl>
        <bibl xml:id="b15"><author><forename><abbr>W.</abbr></forename> <surname>Serauky</surname></author>, <title>Die Affekten-Metrik des Isaac Vossius in ihrem Einfluß auf Joh. Kuhnau und Joh. Seb. Bach</title>, in: <title rend="roman">Fs. M. Schneider</title>, <pubPlace>Lpz.</pubPlace> <date>1955</date>, <biblScope unit="pp">105–113</biblScope></bibl>
      • in: <title rend="roman"> <date>
        <bibl xml:id="b30"><author><forename><abbr>W.</abbr></forename> <surname>Blankenburg</surname></author>, <title>Überlieferung und Textgesch. von Martin Luthers <q>Encomion musices</q></title>, in: <title rend="roman">Luther-Jb.</title> <date>1972</date>, <biblScope unit="pp">80–104</biblScope></bibl>

    22. If a <date> follows directly after <title rend="roman"> without any punctuation in between, tag it as <date> if it is the only date in  the <bibl>, as in this example:

      <bibl xml:id="b51"><author><forename><abbr>H.</abbr></forename> <surname>Kretzschmar</surname></author>, <title>Allgemeines und Besonderes zur Affektenlehre</title>, in: <title rend="roman">JbP</title> <date>1911</date>/<biblScope unit="vol">12</biblScope></bibl>

      Otherwise the dates should be included in <title rend="roman">. In the following example, there is already a date at the end of the citation, so the date is included in the <title rend="roman">:

      <bibl xml:id="b16"><author><forename><abbr>H.</abbr></forename> <surname>Edlén</surname></author>, <title>Musik i Göteborg</title>, in: <title rend="roman">Svenska musikperspektiv. Minnesskrift vid Kungliga Musikaliska Akademiens 200-årsjubileum 1971</title>, <pubPlace>Stockholm</pubPlace> <date>1971</date></bibl>

      A further example:

      <bibl xml:id="b18"><author><forename><abbr>A.</abbr></forename> <surname>Fritz</surname></author>, <title>Die Entwicklung der Aachener Stadtmusik vom städt. Harmoniekorps zum städt. Orch.     25. The abbreviation "Fs.", "Gs." (and the word "Gedenkschrift")  are never abbreviations for a name, they are part of a <title rend="roman>. Note that there is usually a name that follows it, but the whole thing should either be tagged <title rend="roman">Fs. H. Besseler</title> (or <title rend="roman">Capella antiqua München. Fs. zum 25jährigen Bestehen</title> ), or left untagged.

    26. A new instruction: any editor's name that appears in small-caps should be tagged <editor subtype="primary">.

    28. In general, things that don't conform to any of these patterns should not be tagged. In the following example, the words "Vorwort zu" do not fit into any of the patterns, so they should not be tagged.

      <author>M. Seiffert</author>, Vorwort zu <title>J. Ph. Krieger (1649-1725). 21 ausgewählte Kirchenkompos.</title>, <pubPlace>ebd.</pubPlace> <date>1916</date> (= <series>DDT<series> <biblScope unit="vol">53/54</biblScope>)

      Also, if you are at all uncertain, just leave tagging out. The following is thus acceptable:

      <bibl xml:id="b18"><author><forename><abbr>K.</abbr></forename> <surname>Bradley</surname></author>, <title>Gum Leaf Music in Australia</title>, Referat beim Kongreß der ICTM, Canberra, 5.-11. Jan. 1995</bibl>

      The only essential tags are those that relate to rendering (italics, superscript, subscript, smallcaps)

    30. In works lists, when "in:" is followed by "RISM" or "an abbreviation that includes a dash, such as "US-NYp" or "E-Barcelona", do not tag these parts.

    32. <incipit> should only be used in works lists <div type="works">, and never within a <title> element
