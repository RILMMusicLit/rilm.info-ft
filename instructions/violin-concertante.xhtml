The 20th century violin concertante

  1. General comments:
    • This publication is to be converted from PDF to XML.
    • In the PDF pages, an entry comprises a "card" with labeled fields. There is a separate entry for each work title, but ordered according to personal name. Furthermore, the name is repeated for each entry associated with it. In the output files, these should be COMBINED into one entry for each personal name.
    • All entries in the output TEI will therefore be <text type="person">.
    • The pages have no "printed" numbers. Use the PDF file's page number for the TEI Header <biblScope> element.
  2. FRONT MATTER
    • Cover: treat as a title page and create a TEI file for it.
    • Copyright page
    • Preface
    • Elaboration of terms and concepts and A word about references.
      • Combine into ONE TEI file
      • Create an image file for the sample entry on p. 5
  3. ENTRIES Are formatted as "cards".
    • Three boxes at the top are unlabeled, and contain, from top to bottom:
      • Composer's name in non-Latin characters, where appropriate;
      • Composer's name in common or Latinized form, and birth and death dates;
      • Work title.
    • (Subsequent boxes are labeled.)
    • Map the "fields" in the PDF entry as follows:
      • Use the composer's common name and birth and death dates as the <title> in the TEI header, and in <div type="heading">
      • IF there is a non-Latin version of the composers name, use it in <div type="subheading">; else omit the subheading.
      • Create a <div type="works"><list> for the remaining data. If a field contains only a "?", insert an empty element only and omit the label for the field, unless otherwise indicated below.
        • Use the work title for <title>. Add a period after the closing tag.
        • If there is an opus number, include the word "Opus ", no colon, followed by the numeral in <opus> tags. Add a period after the closing tag.
        • Use "year" for the contents of a <date> element. Add parentheses before and after the tags.
        • Use "duration" next. Capitalize the label; do not tag the content; If the value is a "?", omit entirely
        • For "instr", Capitalize the label, tag the content <instrumentation>. Add a period after the tag.
        • If there is content in "mvt", include it after instrumentation; Omit the "mvt" label. Do not tag the content, but insert a semicolon instead of line breaks, and a period at the end. If the content is "?" omit entirely.
        • For "sco", spell out the label "Score: "; Tag the content as <publisher>; add a period after the closing tag.
        • For "wp", spell out the label as "World premiere: "; tag <date>, <rilmPerfomance> and names as shown in the examples. The second-to-last item within the parentheses will be <orgName type="performing">; others will be tagged <persName>.
        • If there is content in "com" other than "–", tag it as <note> and omit the "com" label.
    • If the entry has a "headphone" icon, add an empty <gap n="sound"/> element after the closing </bibl> tag.
    • If the entry has a "musical notes" icon, add an empty <gap n="score"/> element after the closing </bibl> tag.
    • Where there is more than one entry for a composer, combine them into a single TEI file by adding an additional <item> element in the <div type="works"><list> for each entry.
