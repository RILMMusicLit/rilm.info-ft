Musical Instruments: A Comprehensive Dictionary (Marcuse)

This title will be converted from scanned print. Please create a whole-volume PDF as well as article-level TEI and METS-ALTO.

All entries will be <text type="instrument">. There is a lot of information contained in the typography itself, including translations, references, and identification of languages.

  1. Front Matter:
    • Title page (omit the half-title)
    • Copyright page
    • Preface. Include the "Use of the dictionary" as a div type="section".
    • Remaining front matter, including Contents, List of Illustrations, Pitch Names, and Abbreviations
  2. References in general: This work has numerous references between entries that come in several different forms.
    • See-references appear in several forms:
      1. Term1, see: term2. [e.g. Crescent, see: Jingling Johnnie.]
      2. Term1, Lang.: term2 [e.g. Crécelle, Fr.: cog rattle.]
      3. Term2, var. of term2 [e.g. Dromslade, var. of drumslade]
      4. Term2, syn. of term2 [e.g. Dudi, syn. of edaka.]
      Do not make a TEI file for these entries; instead, include the heading term in the <keywords> element of the target entry (refer to the general rules for see references ).
      N.B. If there is additional information, such as a translation, DO make a TEI file and tag the target term with <ref> if possible: [Piob mor (Gaelic: great pipe), see: Highland pipe.]
    • Cross-references within the body text are marked by asterisks. Tag the word following the asterisk with <ref>. If you cannot match on one word, the target entry might be two words:
      [...and in England the so-called *whittle and dub accompanied morris dances...]
      ......and in England the so-called <ref target="??">whittle and dub</ref> accompanied morris dances...
      (There is no entry for "Whittle", but there is for "Whittle and dub".)
    • Cross-references at the end of articles are marked "See also: …" or "See: …" or "Cf. …", with each target separated by a comma.
  3. Dictionary entries titles are in bold, capitalized.
    • Capture the bold-type heading as the <title> in the <teiHeader>. Use the same text for the <div type="heading">, and tag it as <term>. Omit the comma in both cases if it immediately follows the heading.
    • Begin the <div type="body"> with whatever follows the bold type.
      1. Information in parentheses give the language of the term, sometimes followed by an English equivalent in ROMAN. If the English is given, tag the language of the heading term as well as its translation, and include a @corresp attribute:
        [Abeng (Twi: animal horn), 1. generic name in Twi...]
        <div type="heading"><p><term xml:id="t1" xml:lang="tw">Abeng</term></p></div>
        <div type="body"><p>(Twi: <term xml:id="t2" xml:lang="en" corresp="t1">animal horn</term>), 1. generic name in Twi...</p></div>
        If the term(s) following the language is(are) ITALIC, tag the language of the ITALIC term(s):
        [Action (Fr.: mécanique; Ger.: Mechanik; It: meccanica), 1. the intermediary...]
        <div type="heading"><p><term xml:id="t1">Action</term></p></div>
        <div type="body"><p>(Fr.: <term xml:id="t2" corresp="t1" xml:lang="fr">mécanique</term>; Ger.: <term xml:id="t3" corresp="t1" xml:lang="de">Mechanik</term>; It: <term xml:id="t4" corresp="t1" xml:lang="it">meccanica</term>), 1. the intermediary...]</p></div>

        If ONLY the language is given in parenthesis, include it as the @xml:lang of the TERM IN THE HEADING:
        [Abuba (Heb.), reedpipe of ancient Israel...]
        <div type="heading"><p><term xml:id="t1" xml:lang="he">Abuba</term></p></div>
        <div type="body"><p>(Heb.), reedpipe of ancient Israel...</p></div>
      2. Square brackets denote etymology. No need to tag the contents:
        [Aballa [Sp. aballar, to move], …]
        <div type="body"><p>[Sp. <hi rend="italic">aballar</hi>, to move],...</p></div>
  4. Alternate meanings for the same term are marked by bold-type numbers in the body text.
    • Start a new <p> at each number, even though they are inline in the print:
      [Okuelé, 1. Cuban name of the okpelé; 2. Afro-Cuban ritual drum, goblet-shaped...]
      <div type="heading"><p><term xml:id="t1">Okuelé</term></p></div>
      <div type="body"><p>1. Cuban name of the <ref>okpelé</ref>;</p>
      <p>2. Afro-Cuban ritual drum, goblet-shaped...</p></div>
  5. Bibliographical references are given as numerals, in parentheses, that are keyed to the bibliography at the end of the volume (beginning page 603). Include a <div type="ography"> and replace each number in the entries with a <bibl> element containing the full marked-up bibliographic item from pages 603-608. See the example files.
  6. Photograph plates: make a separate xml file for each; file names as "mcd-plate01" through "mcd-plate24".

Please also refer to the example files.
