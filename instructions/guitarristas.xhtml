Diccionario de Guitarristas... (pdg)

  1. General comments:
    • Convert from the PDF created by LM Tech.
    • Note that the text layer is NOT very accurate. OCR should be redone with a Spanish dictionary.
    • The first 17 pages of the PDF will not be used.
    • Cross-references (See-references) are in the form LASTNAME, Firstname — "Ver" Lastname2 Firstname2. Treat these according to http://rilm.info/ft/tei_rilm_instructions.html#see-refs .
    • Convert all quotation marks ( “ ” ) to <q> tags.
  2. SSA Front matter
    • Portrait of the author, use "[Frontispiece]"
      • for the title.
    • Title page
    • OMIT the Numeration and signature page (p.20 of the PDF).
    • Preludio
  3. PERSON ENTRIES (print pp. 11-368) begin with bold, all-caps headings; formatted as LASTNAME, Forename. —
    • To create the <title> element in the TEI Header, omit the final period unless there is an abbreviation:
      CASTILLO, David del. —
      <titleStmt><title>CASTILLO, David del</title></titleStmt>
      CATALA, A. F. —
      <titleStmt><title>CATALA, A. F.</title></titleStmt>
    • Use the same content for <div type="header">, but keep the final period. Omit the M-dash.
      <div type="heading" xml:id="d1"><p xml:id="p1"><rilmPerson xml:id="r1"><persName><surname>CASTILLO</surname>, <forename>David</forename> <nameLink>del</nameLink></persName></rilmPerson>.</p> </div>
    • Where a forename is unknown, it is indicated in the print by "(…)". Omit this for the TEI Header, but include it in the <div type="heading">, tagged as <forename>:
      CATALDI ( ... )
      <titleStmt><title>CATALDI</title></titleStmt>
      <div type="heading" xml:id="d1"><p xml:id="p1"><rilmPerson xml:id="r1"><persName><surname>CATALDI</surname> <forename>( … )</forename></persName></rilmPerson></p></div>
    • Start <div type="body"> for the paragraph(s) of prose.
    • Many Person entries contain indented lines of poetry. Tag with <lg> and <l> tags. Omit surrounding quotation marks if present, but keep quotation marks within:
      "Quando nascen Passarinho
      Trouxe quatro dote junto:
      Ser branco, dar-se a respeíto,
      "Tocar pouco" e cantar muíto."
      <lg><l>Quando nascen Passarinho</l>
      <l>Trouxe quatro dote junto:</l>
      <l>Ser branco, dar-se a respeíto,</l>
      <l><q><hi rend="bold">Tocar pouco</hi></q> e cantar muíto.</l></lg>
    • There are no worklists nor bibliographies per se in the person entries. Tag things that look like tables (that clearly are arranged in columns) as <table> (e.g. on p.25, 50); things that look like lists (that don't have clear columns) as <list> (e.g. on p.63, pp.79-82).
    • A few entries have an author given in bold at the end, right justified. (See print p.143 for example.) Tag with <div type="authors"> and <docAuthor>.
    • Apendice de guitarristas (pp. 361-368): Process identically to entries in the main Persons section.
    • Luthiers (pp. 369-410): Process as preceding sections, EXCEPT add "[Luthier]" to the <title> element content in the TEI header.
    • I have found two TOPIC entries withing the names: ACTORES GUITARRISTAS (p.12), and POSEEDORES DE GUITARRAS "TORRES"(p.392). There might be more that I have not found. They are distinguished by NOT having an M-dash following the heading. For handling, see below "Topic entries".
  4. TOPIC ENTRIES (pp. 399-464):
    • Instruments (pp. 399-422): Tag as <text type="instrument">. Omit the trailing period and M-dash from the <title> and <div type="heading">. Tag the heading as <term>. Where this is a lower case "o" in the heading (meaning "or"), tag the terms before and after separately:
      SHARUD o SHORODE.—
      <titleStmt><title>SHARUD o SHORODE</title></titleStmt>
      <div type="heading" xml:id="d1"><p xml:id="p1"><term xml:id="t1">SHARUD</term> o <term xml:id="t2">SHORODE</term></p></div>
    • Danzas (pp. 425-452): Process as above, except <text type="genre">.
    • Terminologia (pp. 455-464): Process as above, except <text type="topic">.
